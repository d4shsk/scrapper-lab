## Описание задания

Необходимо выполнить скрапинг страниц Электронного атласа РГПУ ([atlas.herzen.spb.ru](https://atlas.herzen.spb.ru)) и сформировать CSV-файл со следующими полями:

| Поле      | Описание                          |
|-----------|-----------------------------------|
| `ФИО`     | Полное имя преподавателя          |
| `Почта`   | E-mail (если указан в профиле)    |
| `Телефон` | Номер телефона (если указан)      |

### Порядок выполнения

1. Обойти страницы со списком преподавателей (54 страницы):
   `https://atlas.herzen.spb.ru/teachers?page=1` … `?page=54`
2. С каждой страницы собрать ФИО и ссылку на профиль преподавателя.
3. Зайти на каждую страницу профиля и извлечь почту и телефон (при наличии).
4. Сохранить всё в CSV-файл.

## Анализ структуры сайта

Сайт работает на **Laravel + Livewire**. Важная особенность: первоначальный HTML рендерится **на сервере** (SSR), поэтому данные доступны в исходном коде страницы без выполнения JavaScript.

### Страница со списком (`/teachers?page=N`)

Каждый преподаватель представлен строкой таблицы:

```html
<tr class="bg-gray-100">
  <td class="border-t-0 px-6 ...">
    <a href="https://atlas.herzen.spb.ru/teachers/10417" class="text-blue-600">
      Абаза Дарья Александровна
    </a>
  </td>
  <td ...>Доцент</td>
  <td ...>Институт музыки, театра и хореографии</td>
  ...
</tr>
```

**CSS-селектор для ссылок:** `td a.text-blue-600`

### Страница профиля (`/teachers/{id}`)

E-mail и телефон находятся в блоках с иконками:

```html
<!-- Email -->
<div class="flex items-center text-blue-400 py-1 rounded-lg cursor-pointer mb-1">
  <svg><!-- иконка конверта --></svg>
  <h1 class="text-m">zerebin@mail.ru</h1>
</div>

<!-- Телефон (если есть) -->
<div class="flex items-center text-blue-400 py-1 rounded-lg cursor-pointer mb-1">
  <svg><!-- иконка телефона --></svg>
  <h1 class="text-m">+7 (812) 123-45-67</h1>
</div>
```

**Стратегия извлечения:**
- Email — содержит символ `@`
- Телефон — содержит символ `+` или цифровой паттерн

Оба поля находятся в элементах `h1.text-m` внутри блоков `.flex.items-center.text-blue-400`.

## Способ 1: Python + BeautifulSoup

### Установка зависимостей

```bash
pip install requests beautifulsoup4 lxml
```

### Код

Код скрапера расположен в файле `scraper_bs4.py`.

Скрипт выполняет три шага:
1. Обходит страницы с 1 по 54, собирает ФИО и ссылки на профили.
2. Переходит на каждый профиль и извлекает почту и телефон.
3. Сохраняет результат в `teachers_bs4.csv`.

### Запуск

```bash
python scraper_bs4.py
```

### Ожидаемый вывод

```
=== Шаг 1: Сбор списка преподавателей (страницы 1–54) ===
  Страница 1/54... найдено: 20
  Страница 2/54... найдено: 20
  ...
Всего преподавателей: ~1080

=== Шаг 2: Сбор контактных данных ===
  [1/1080] Абаза Дарья Александровна
  [2/1080] Абаза Федор Викторович
  ...

=== Шаг 3: Сохранение в teachers_bs4.csv ===
  Записей: 1080, с контактами: 650
  Файл сохранён: teachers_bs4.csv
```

### Формат результирующего CSV

```csv
ФИО,Почта,Телефон
Абаза Дарья Александровна,zerebin@mail.ru,
Абаза Федор Викторович,,
Абашина Анна Дмитриевна,abashina@herzen.spb.ru,+7 (812) 315-63-40
```

### Примечания к реализации

| Аспект | Детали |
|--------|--------|
| **Парсер** | `lxml` — быстрее встроенного `html.parser` |
| **Задержка** | 0.5 сек между запросами — не перегружает сервер |
| **Кодировка** | `utf-8-sig` (BOM) — корректно открывается в Excel |
| **Дубликаты** | Возможны (одинаковое ФИО, разные ID) — сохраняются все |
| **User-Agent** | Имитация браузера для обхода простых блокировок |

## Способ 2: MCP-сервер Puppeteer

**Puppeteer** — это Node.js библиотека для управления браузером Chromium. В отличие от BeautifulSoup, Puppeteer выполняет JavaScript и видит итоговую DOM после всех динамических обновлений.

В данном подходе MCP-сервер Puppeteer **подключается к ИИ-ассистенту** (например, SourceCraft Code Assist), предоставляя ему инструменты для управления браузером: `puppeteer_navigate`, `puppeteer_evaluate`, `puppeteer_screenshot`. Ассистент самостоятельно открывает страницы, извлекает данные и формирует CSV.

### Конфигурация MCP-сервера

В файле `mcp.json` добавить:

```json
{
  "mcpServers": {
    "puppeteer": {
      "command": "npx",
      "args": ["-y", "puppeteer-mcp-server"],
      "env": {},
      "disabled": false
    }
  }
}
```

После перезапуска ИИ агенту инструменты Puppeteer станут доступны ассистенту автоматически.

### Промт для агента

Вместо пошагового взаимодействия отправьте агенту один полный промт — он самостоятельно пройдёт все шаги без уточняющих вопросов:

```
Используя puppeteer, выполни следующие шаги:

1. Обойди страницы https://atlas.herzen.spb.ru/teachers?page=1
   по https://atlas.herzen.spb.ru/teachers?page=54.
   На каждой странице выполни скрипт и собери ссылки на профили:
   (() => {
     return JSON.stringify(
       Array.from(document.querySelectorAll("td a.text-blue-600[href*='/teachers/']"))
         .map(a => ({ name: a.textContent.trim(), url: a.href }))
     );
   })()

2. Для каждой собранной ссылки перейди на страницу профиля
   и выполни скрипт для извлечения контактов:
   (() => {
     const contacts = { email: "", phone: "" };
     document.querySelectorAll("h1.text-m").forEach(el => {
       const text = el.textContent.trim();
       if (text.includes("@")) contacts.email = text;
       else if (/^[\d\s\+\-\(\)\.]+$/.test(text) && text.length > 5) contacts.phone = text;
     });
     return JSON.stringify(contacts);
   })()

3. Собери все данные в массив {ФИО, Почта, Телефон}
   и сохрани результат в файл teachers_puppeteer.csv
   с заголовком: ФИО,Почта,Телефон
```

## Сравнение подходов

| Критерий | Python + BeautifulSoup | MCP Puppeteer |
|---|---|---|
| **Скорость** | Быстро (только HTTP-запросы) | Медленнее (полный браузер) |
| **JS-рендеринг** | Не поддерживается | Полная поддержка |
| **Простота** | Лаконичный код | Управление через ИИ, не требует писать скрипт |
| **Ресурсы** | Минимальные | Требует Chromium (~300 МБ) |
| **Применимость** | Для данного сайта (SSR) | Для любого сайта, включая SPA |
| **Обход защит** | Уязвим к JS-проверкам | Имитирует реальный браузер |

### Вывод

Для данного сайта ([atlas.herzen.spb.ru](https://atlas.herzen.spb.ru)) оба метода работают корректно, так как Livewire выполняет SSR. Python + BeautifulSoup предпочтительнее для самостоятельного запуска: проще в освоении и быстрее выполняется. Puppeteer через MCP полезен, если сайт перейдёт на клиентский рендеринг или добавит защиту от ботов.
